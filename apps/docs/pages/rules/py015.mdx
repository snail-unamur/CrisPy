## <BadgeQuestionMark className="icon"/> Pourquoi ?

Créer beaucoup de petits objets disperse les données en mémoire et réduit l’efficacité du cache.

## <BadgeX   className="icon"/> Mauvaise pratique

```python
data = []
for i in range(100000):
    data.append([i])
```

Explication :
Création de milliers de petits objets → fragmentation logique.

## <MessageCircleCheck  className="icon"/> Bonne pratique

```python
data = [i for i in range(100000)]
```

Explication :
Structure compacte → meilleure localité mémoire.

## <GitCompareArrows  className="icon"/> Comparaison avec le langage C

```c
int** data = malloc(n * sizeof(int*));
```

Multiples allocations → fragmentation du heap.

## <GitCompareArrows  className="icon"/> Comparaison avec Java

```java
List<List<Integer>> data = new ArrayList<>();
```

NB: Beaucoup d’objets → surcharge mémoire et mauvaise localité.

```java
int[] data = new int[n];
```

Structure compacte → meilleure performance.

## <Zap  className="icon"/> Performance

 - Meilleure utilisation du cache

 - Moins d’allocations mémoire

 - Exécution plus rapide
