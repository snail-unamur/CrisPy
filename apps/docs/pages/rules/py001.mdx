## <BadgeQuestionMark className="icon"/> Pourquoi ?

Les tableaux NumPy permettent d'effectuer des opérations vectorisées et offrent une meilleure efficacité en termes de gestion de la mémoire.

## <BadgeX   className="icon"/> Mauvaise pratique

```python
arr = [1, 2, 3]
```

Explication : En Python standard, une liste est flexible mais lente pour les calculs mathématiques car elle peut contenir n'importe quel type d'objet, ce qui consomme plus de ressources.

## <MessageCircleCheck  className="icon"/> Bonne pratique

```python
import numpy as np
arr = np.array([1, 2, 3])
```

Explication : En utilisant NumPy, vous stockez les données de manière compacte. Cela permet d'effectuer des calculs sur l'ensemble du tableau d'un seul coup (vectorisation) sans utiliser de boucles for explicites.

## <GitCompareArrows  className="icon"/> Comparaison avec C

Comment cela fonctionne :
Par exemple, en C, un tableau ne contient que des éléments du même type de données, ce qui est exactement la philosophie adoptée par NumPy pour gagner en rapidité.

```c
int numbers[5] = {1, 2, 3, 4, 5};
```

## <Zap  className="icon"/> Performance

L'utilisation de listes Python pour des calculs intensifs est un "anti-pattern" (une mauvaise habitude).
Les listes obligent Python à vérifier le type de chaque élément à chaque opération, alors que NumPy traite les données de manière groupée, ce qui est souvent des dizaines de fois plus rapide.
