# PY036 – Éviter la croissance mémoire non contrôlée

## Pourquoi ?

Accumuler des données inutilement entraîne une consommation mémoire excessive.

## Mauvaise pratique

```python
cache = []

for i in range(1000000):
    cache.append(i)
```

Explication :
La structure grandit sans limite → utilisation mémoire inutile.

## Bonne pratique

```python
for i in range(1000000):
    process(i)
```

Explication :
On traite les données sans les stocker inutilement.

## Comparaison avec le langage C

```c
int* arr = malloc(1000000 * sizeof(int));
free(arr);
```

La mémoire doit être allouée et libérée manuellement.

## Comparaison avec Java

```java
List<Integer> list = new ArrayList<>();

for(int i = 0; i < 1000000; i++) {
    list.add(i);
}
```

Même problème qu’en Python :

 - croissance mémoire

 - pression sur le Garbage Collector

## Performance

 - Réduction de la mémoire utilisée

 - Moins de pression sur le GC

 - Meilleure stabilité du programme
