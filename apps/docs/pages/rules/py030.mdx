# PY030 – Réduire le coût des objets Python

## Pourquoi ?

Chaque objet Python contient des métadonnées (type, référence, etc.), ce qui augmente fortement la mémoire utilisée.

## Mauvaise pratique

```python
numbers = [1, 2, 3, 4, 5]

```
Explication :
Chaque entier est un objet Python → overhead mémoire important.

## Bonne pratique
```python
import numpy as np

numbers = np.array([1, 2, 3, 4, 5]
)
```

Explication :
NumPy utilise des types compacts en mémoire, similaires à ceux du langage C.

## Comparaison avec le langage C
```c
int numbers[5] = {1, 2, 3, 4, 5};
```

Aucun overhead → mémoire minimale.

## Comparaison avec Java

Mauvaise pratique
```java
Integer[] arr = {1, 2, 3, 4, 5}; // Mauvais
```

Bonne pratique
```java
int[] arr = {1, 2, 3, 4, 5}; // Bon
```

Integer est un objet (comme Python), tandis que int est un type primitif plus léger.

## Performance

 - Réduction de la mémoire utilisée

 - Meilleure utilisation du cache CPU

 - Traitements plus rapides
